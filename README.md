# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
* リモートリポジトリとは
リモートリポジトリとは「git hub」などネット上にファイルやディレクトリの状態を保存・管理することができる場所です。

*ローカルリポジトリとは 
ローカルリポジトリとは「git」などPC内でファイルやディレクトリの状態を保存・管理することができる場所です。


## プッシュとマージの違いは何でしょうか？
*プッシュ「push」
プッシュはローカルリポジトリからリモートリポジトリへ送信する作業です。

*マージ「merge」
マージとは分岐して作業していたファイルやデータを融合させる作業です。

プッシュはローカルリポジトリからリモートリポジトリへ送信する作業なのに対し、マージは分岐して作業していたデータを融合させる作業になります。

## コミットとプッシュの違い
*コミット
ファイルやデータをコメントを残し作業工程（現在のデータの状態）を保存する作業。

*プッシュ「push」
プッシュはローカルリポジトリからリモートリポジトリへ送信する作業です。

## コミットのメッセージはどのように書いてあげるのが適切でしょうか？
コミットのメッセージは作業の進捗状況（何を作成したのか）や修正内容などを具体的に記載し、見返したときにすぐわかるよう記載する。


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
ローカルでマージする際はマージする前に他エンジニアにデータを確認してもらうことが難しいのに対し、プルリクエストでマージする際は、一度リモートリポジトリにプッシュし他の開発メンバーが確認してからマージすることでミスを防ぎより正確な状態でマージすることが可能である。


## コンフリクトを起こしてしまった場合、どう対処すべきですか？
状況により変わりますが、先にマージされた変更内容を取り込む場合と後にマージされた変更内容を取り込む場合、また注意が必要ですが両方を取り込む場合があります。
